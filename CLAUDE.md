## Runtime

Use Bun, not Node.js. `bun test` for tests, `bun run src/cli.ts` to run the CLI.

## Architecture

- `src/cli.ts` — Entry point. Parses flags, resolves faker instance, dispatches to handlers.
- `src/resolver.ts` — Resolves `"module.method"` strings to faker calls. Validates module/method names.
- `src/schema.ts` — Recursively resolves faker method references in JSON schema templates.
- `src/serialize.ts` — Converts any faker return value to JSON. Handles nested BigInt/Date.
- `src/enumerate.ts` — Lists modules/methods via prototype chain walk. Filters callback-only helpers.
- `src/describe.ts` — Module/method descriptions from `src/metadata.json`. Formats for CLI output.
- `src/format.ts` — TTY-aware column and table formatting for `--list`.
- `src/metadata.json` — Static method metadata (params, types, descriptions, examples) generated from faker's `.d.ts`.

## Testing

`bun test` runs all tests. Test files are in `test/`.

- `test/fixtures/return-types.json` — 240 deterministic fixture entries (ground truth for all return types). Generated by `test/fixtures/generate.ts` with seed 12345 and ref-date 2025-01-01.
- `test/fixtures/generate-metadata.ts` — Parses faker's `.d.ts` to produce `src/metadata.json`. Run with `bun run generate-metadata`.

## Key conventions

- All CLI output is valid JSON to stdout. Errors go to stderr.
- Error messages should guide the user to the next action (e.g. "Run: faker --list").
- The `helpers` module is enabled but callback-requiring methods (`unique`, `maybe`, `multiple`) are excluded.
- `serialize()` must handle BigInt and Date at any nesting depth.
- Fixtures and metadata are committed (not generated at build time).

## Build & release

- `bun run build` compiles standalone binaries for 5 platforms via `bun build --compile`.
- `/release` command handles version bumps, changelog, and push. The GitHub Actions workflow creates the release.
